Introduction¶

Analyzing Passenger Satisfaction in Airline Services: An Exploratory Data Analysis Approach

Introduction: Understanding customer satisfaction is a critical factor in the airline industry, as it directly impacts brand loyalty, reputation, and revenue. This project performs an in-depth exploratory data analysis (EDA) on a commercial airline passenger dataset to uncover the key factors that influence satisfaction levels.

The dataset contains both categorical and numerical features, including service quality metrics (e.g., seat comfort, online boarding), travel details (e.g., flight class, travel type), and delays. Through visualizations and statistical insights, we explore how these variables interact with the satisfaction label, identify trends, assess feature importance, and detect data imbalances or anomalies.

This analysis serves as a foundation for building predictive models and making data-driven recommendations for improving customer experience in the airline industry.

  1. Median Differences:

If the median line (horizontal line in the box) for the “satisfied” group is consistently higher, it suggests that higher ratings in that feature are associated with satisfaction.

  1. Distribution Spread:

Wider boxes indicate more variability in how customers rate that feature. If dissatisfied customers have a wider box and lower medians, that feature may need improvement.

  1. Outliers:

Outliers (dots outside whiskers) can show individual unusual experiences — e.g., some satisfied customers may still rate a service poorly, suggesting tolerance or variability in expectations.

  1. Feature Importance (Visually):

If a feature shows a large gap in median rating between satisfied and dissatisfied groups (e.g., "Inflight entertainment"), it could be a key driver of satisfaction. Features with similar distributions across both groups may be less influential.

# Initialize results storage
results_df = {
    'Model': [],
    'Accuracy': [],
    'F1-Score': []
}
def evaluate_model(model, model_name, X_test, y_test):
    y_pred = model.predict(X_test)
    accuracy = accuracy_score(y_test, y_pred)
    f1 = f1_score(y_test, y_pred, average='weighted')  # because its binary classification
    
    results_df['Model'].append(model_name)
    results_df['Accuracy'].append(round(accuracy,4))
    results_df['F1-Score'].append(round(f1,4))

Conclusions¶

Regarding hyperparamter tunning, these are the optimal values:

  • Logistic Regression: {'C': 1, 'max_iter': 100, 'penalty': 'l2', 'solver': 'sag'}
  • Random Forest: {'max_depth': 30, 'min_samples_leaf': 1, 'min_samples_split': 10, 'n_estimators': 100}
  • Adaboost: {'algorithm': 'SAMME.R', 'estimator__max_depth': 3, 'learning_rate': 1.0, 'n_estimators': 100}

To sum up:

  • Logistic Regression, while interpretable, underperformed (87.1% accuracy) due to its linearity assumption, making it unsuitable as a final solution.

  • Random Forest is the top performer, achieving 96.2% accuracy and near-perfect F1-scores (0.96–0.97) for both satisfied and dissatisfied passengers. This model is ideal for deployment because it is the best at capturing complex feature interactions.

  • AdaBoost is a strong alternative, with marginally lower accuracy (95.6%) but faster training. So, if there is a focus on speed and lightweight deployment, Adaboost will be a better option.
